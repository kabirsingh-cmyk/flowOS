01 — The system

Seven layers. One governed loop.

Strategy, context, planning, execution, intelligence, integration, and audit. Every externally visible action passes through policy before it ships.

01 / Context
Brand memory that holds the line
Durable vocabulary, values, claims, and prohibited territory. Versioned. Approval-gated. Brand-scoped.
02 / Voice
Dynamic tone modes
Two to five modes per brand, each with rhythm, vocabulary, and examples. Assigned per campaign, validated per draft.
03 / Execution
Copy, email, and AI creative in one queue
Captions, subject lines, UGC-style scripts, HeyGen videos, fal.ai renders — all drafted, all reviewable, all logged.
04 / Governance
Policy Guard in front of every send
Allow, revise, or require-approval — with the reasoning attached. Hard boundaries are non-negotiable.

04 — Governance

Autonomy is a dial, not a switch.

Operating principle
Strategy, paid spend, crisis responses, core brand memory, and AI video all require human approval — regardless of mode.

Strategic, financial, legal, reputational, and crisis-sensitive actions never ship themselves. Everything else is a configuration, not a code change.

MODE 01
Manual / Copilot
Drafts and recommendations only. Nothing leaves the building without you.
default
MODE 02
Assisted Autonomous
Low-risk tasks run automatically. High-risk routes to approval.
recommended
MODE 03
Autonomous with Guardrails
Approved patterns publish within configured limits. Audit everything.
phase 2
Hard boundaries
  • — Strategy changes
  • — Paid spend changes
  • — Crisis, legal, or PR responses
  • — Core brand memory edits
  • — New tone mode creation
  • — AI-generated video asset publish
  • — Anything the owner configures as high-risk
